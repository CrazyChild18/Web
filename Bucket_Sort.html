Theory:
Bucket sorting, as the name implies, is to put data into several ordered buckets, sort the data in each bucket, and finally take out the data in each bucket in order from small to large, that is, the sorting is completed.
Tips：
1.The number of buckets is determined by the number of buckets in this group.
2.The subscript of the bucket is determined by (number * array length/(maximum number +1))
Complexity:
If we have n pieces of data to sort, we divide them into m buckets, so that the data in each bucket is k = n/m.The time complexity of sorting in buckets is order k logk.M barrels is m * O (k * logk) = m * O (log (n/m) * (n/m)) = O (nlog (n/m)).Log of n over m is a smaller constant when the number of buckets is close to the number of data, so the time complexity is close to order n
Application scenarios:
Although the time complexity of bucket sorting is linear, it is not a substitute for sorting algorithms with time complexity of O(nlogn) such as quicksort and merge.Because the bucket sorting scenario is very rigorous: first, the data should be evenly distributed.In the worst case, if all the data is in a bucket, does the time complexity of bucket sorting degenerate to O(nlogn)?Second, the data to be sorted should easily be divided into m buckets, and each bucket should also have a size order.
